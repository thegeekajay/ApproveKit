Why teams need it

Agents should not get production power without a checkpoint.

ApproveKit makes human review a reusable middleware capability instead of one-off approval logic scattered through every tool.

Stop unsafe execution

Rejected and timed-out requests never run the wrapped tool body. The default failure mode is deny.

Show the reviewer context

Risk labels, arguments, notes, timestamps, and policy metadata stay attached to each request.

Keep audit history

Every approval, rejection, timeout, and auto-approved call is persisted for local inspection.

How it works

A small policy gate in the tool execution path.

1

Agent calls a guarded tool

The `@kit.guard` decorator captures tool name and arguments before execution.

2

Policy decides approval mode

Safe tools execute immediately. Risky tools become pending approval requests.

3

Reviewer approves or rejects

The browser reviewer shows redacted payloads, risk labels, and decision notes.

4

Audit is recorded

Approved calls execute. Rejected and timed-out calls are blocked and logged.

Run it locally

Two terminals are enough for the MVP demo.

The demo creates a shared SQLite database. The agent waits while the reviewer makes decisions in the browser.

Terminal 1

python3 -m pip install -e ".[dev]"
python3 demo/agent.py --db /tmp/approvekit_demo.db --reset

Terminal 2

approvekit-web --db /tmp/approvekit_demo.db --port 8765
